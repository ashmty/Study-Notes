Git: is a software that allows you to keep track of changes made to a project over time. Git works by recording the changes you make to a project, storing those changes, then allowing you to reference them as needed, Git stores this information in a data structure called a repository

    1. Git Basics:
    1. git init: to initialize a new git project;in the folder where the project is Cmd : git init.
    2. Git Workflow: i's like having 3 parts
      • A Working Directory: where all the work is done: creating, editing, deleting and organizing files/li>
      • A Staging Area: list chages made.
      • A Repository: store changes as different versions of the project
    3. git status: git status in cmd to check the status of the changes made.
    4. git add: to add files to the staging area , git add. ( add all modified files to the repo), you can add more than one file at the same time ==> git add filename_1 filename_2
    5. git diff: check the differences between the working directory and the staging area
    6. git commit: A commit permanently stores changes from the staging area inside the repository. it must have the option -m followed by a message,Standard Conventions for Commit Messages:
      1. Must be in quotation marks
      2. Written in the present tense
      3. Should be brief (50 characters or less) when using -m
      4. the current commit is called the head commit , and it cab be called useing git show HEAD
    7. git log:to refer back to an earlier version of a project.
    8. git --help: to get help menu from CMD
  2. Backtracking :
    1. git checkout : will restore the file in your working directory to look exactly as it did when you last made a commit.
    2. git reset: This command resets the file in the staging area to be the same as the HEAD commit. It does not discard file changes from the working directory, it just removes them from the staging area. ==> git reset HEAD filename
    3. git reset HEAD: using the first 7 charcters of the SHA , git reset 5d69206 , this will reset the commit to the head with the hash 5d69206
  3. git branch: the main is called the master,the New Branch is a different version of the Git project. It contains commits from Master but also has commits that Master does not have.
    ==> get branch ( give a list of branches)
    1. Create new branch: git branch new_branch ( ps. we can use - or _ bur no spaces allowed)
    2. git checkout branch-name: to switch to the branch from the master while the master stay intact
    3. git merge: to merge branch with the master and commit changes to the master
      ==> git merge brancg-name
    4. Delet git branch: git branch -d branch_name, if they were never merged to the master we have to use -D
    5. commit after conflict: got commit -m 'resolve merge conflict'
  4. git collaboration:
    1. git clone: git clone remote_location clone_name, remote_location tells Git where to go to find the remote. This could be a web address, or a filepath, the remote original is called origin
    2. git remote -v : to get a list of a Git project's remotes,navigate as CMD folder with cd..
    3. git fetch: An easy way to see if changes have been made to the remote and bring the changes down to your local copy, This command will not merge changes from the remote into your local repository. It brings those changes onto what's called a remote branch synatx : enter the clone folder then git fetch
    4. git merge: git merge origin/master
    5. Git workflow:
      1. Fetch and merge changes from the remote
      2. Develop the feature on your branch and commit your work
      3. Fetch and merge from the remote again (in case new commits were made while you were working)
      4. Push your branch up to the remote for review
    6. git push: git push origin your_branch_name
    7. to navigate CMD , cd / pwd: to know where your are
