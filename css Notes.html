CSS Grid

  1. Creating a Grid:Needs grid container and grid items,Parent selector {display: grid;} for Block Elements, or {display: inline-grid;}, There are two axes in a grid layout : the column (or block) axis and the row (or inline) axis.
    • By default the Grid have only 1 column
    • to specify the columns and sizes : {grid-template-columns:10 20 30} for 3 columns the value
    • to specify rows and sizes: {grid-template-rows: 10px 20% 30em }
    • to replace both Values above: {grid-template: 200px 300px ( rows ) / 20% 10% 70% ( columns)}
  2. fr: unit to divide available space between rows and columns in this Example 60px out of 100px leaves 40px divided between 2
  3. Repeat: {grid-template-columns: repeat(3, 100px)} this value will repeat it 3 times, so instaed of writting {grid-template-columns: 100px 100px 100px;}
  4. minmax: for the grid part not to go smaller than a specific size {grid-template-columns: 100px minmax(100px, 500px) 100px;} so the first and the third have a specific size while the one in the middle change with the browser
  5. Grid Gap: allow space between the items in the grid, it will not add space at the beginning or end of the grid {grid-row-gap:}or {grid-column-gap} or {grid-gap:rows column} ps. no / inbetween
  6. Multiple Row Items: Row grid lines and column grid lines start at 1 and end at a value that is 1 greater than the number of rows or columns the grid has
    • grid-row-start and grid-row-end where always the value of end is higher than the needed value eg. rows from 2 to 3 = start 2 , end 4
    • shortcute : grid-row { start / end }
    • Span: till the browser to span over 2 cells, {grid-column: 4 / span 2;} start at 4 and join 2 cells
    • Grid-area: shortcute for spanning rows and cloumns for 1 cell ; {grid-area: 2 grid-row-start/ 3 grid-row-end/ 4 grid-column-start / 5 grid-column-end }
  7. Grid-template-areas: allows you to name sections of your web page to use as values in the grid-row-start, grid-row-end, grid-col-start,grid-col-end, and grid-area
    • Each area is called for how many cells in the grid
    • Each item gets a {grid-area: name for each block}
  8. Overlapping Elements: Css grid allows over lapping of cells , just add z-inedx:5 for the top level.
  9. Justify-items: is a property that positions Grid items along the inline, or row, axis. This means that it positions items from left to right across the web page. Demo
    • start : aligns grid items to the left side of the grid area

Flexbox:Demo

  1. There are two important components to a flexbox layout: flex containers and flex items. A flex container is an element on a page that contains flex items, All direct child elements of a flex container are flex items.
  2. Flex containers are helpful tools for creating websites that respond to changes in screen sizes:Child elements of flex containers will change size and location in response to the size and position of their parent container.
  3. To designate an element as a flex container, {display: flex or inline-flex} then there are several properties we can use to specify how its children behave:
    1. inline-flex: allows us to create flex containers that are also inline elements.
    2. justify-content: space along the main axis ( by default row ) all of the child elements (flex items) move toward the upper left corner of the parent container,justify-content specify how flex items spread out from left to right, along the main axis. syntax:(.container { display: flex; justify-content: flex-end; }
      1. flex-start: all items will be positioned in order starting, from the left of the parent container, with no extra space between or before them.
      2. flex-end:all items will be positioned in order, with the last item starting on the right side of the parent container, with no extra space between or after them.
      3. center: all items will be positioned in order, in the center of the parent container with no extra space before, between, or after them.
      4. space-around:items will be positioned with equal space before and after each item, resulting in double the space between elements.
      5. space-between:items will be positioned with equal space between them, but no extra space before the first or after the last elements.
    3. align-items:to space flex items vertically.( along the cross access top to buttom)
      1. flex-start : all elements will be positioned at the top of the parent container.
      2. flex-end : all elements will be positioned at the bottom of the parent container.
      3. center : the center of all elements will be positioned halfway between the top and bottom of the parent container.
      4. baseline : the bottom of the content (text) of all items will be aligned with each other.
      5. stretch : if possible, the items will stretch from top to bottom of the container (this is the default value; elements with a specified height will not stretch; elements with a minimum height or no height specified will stretch)
    4. align-self just applied to 1 child in the container , syntac flex-child { align-self:flex-start; }
    5. flex-grow:specify if items should grow to fill a container ( the empty space remaing )and also which items should grow proportionally more or less than others., This is a flex items property.{flex-grow: 1 or 2 } 2 stretch twice as much as 1 when the page get resized ( responsiveness)
    6. flex-shrink:same if the page gets smaller takes a value of 1 or 2 as well, min-width and max-width can override., can accept the value of 0 , where the item will shrink faster,flex-basis
    7. flex-basis: to specify the width of an item before it stretches or shrinks,
    8. flex: as a property in the flex-item it will work as a shortcut {flex: grow shrink basis}, Note that there is no way to set only flex-shrink and flex-basis using 2 values; { flex: 1 ; will ditribute items equally in the container }
    9. flex-wrap: allow flex items to move to the next line when necessary (when shrink the page) . syntax {flex-wrap: wrap}
      1. wrap:child elements of a flex container that don't fit into a row will move down to the next line
      2. wrap-reverse:he same functionality as wrap, but the order of rows within a flex container is reversed
      3. nowrap:prevents items from wrapping; this is the default
      4. Align-content: to space the rows from top to bottom after they are wrapped and the accept 6 values:
        1. flex-start:all rows of elements will be positioned at the top of the parent container with no extra space between.
        2. flex-end:all rows of elements will be positioned at the bottom of the parent container with no extra space between
        3. center:all rows of elements will be positioned at the center of the parent element with no extra space between.
        4. space-between:all rows of elements will be spaced evenly from the top to the bottom of the container with no space above the first or below the last.
        5. space-around: all rows of elements will be spaced evenly from the top to the bottom of the container with the same amount of space at the top and bottom and between each element.
        6. stretch:if a minimum height or no height is specified, the rows of elements will stretch to fill the parent container from top to bottom (default value).
        7. to get equal size ( if there is 3 items) we can use width = 33%, if there is margin 10px we can use width: clac(33.33% - 20px) to make equal size
    10. flex-direction:the default is row, flex containers have two axes: a major axis ( left to right )and a cross axis ( top to button ). The major axis covers: the following properties: justify-content,flex-wrap, flex-grow, flex-shrink, The cross axis: align-items,align-content, the flex dirctions are interchangeable , If we add the flex-direction property and give it a value of column, the flex items will be ordered vertically, not horizontally.
        Syntax:L {flex-direction: ----}
      1. row:elements will be positioned from left to right across the parent element starting from the top left corner (default)
      2. row-reverse:elements will be positioned from right to left across the parent element starting from the top right corner.
      3. row:elements will be positioned from top to bottom of the parent element starting from the top left corner.
      4. row:elements will be positioned from the bottom to the top of the parent element starting from the bottom left corner.
    11. flex-flow:shortcut used to declare both the flex-wrap and flex-direction properties in one line, The flex-flow property is declared on flex containers, syntax: {flex-flow : direction wrap } ,
    12. u>Nested Flexboxes::to position flex containers inside of one another.

Media Query


Syntax:
  1. @media:This keyword begins a media query rule and instructs the CSS compiler on how to parse the rest of the rule.
  2. only screen: Indicates what types of devices should use this rule.
  3. media Feature: and (max-width : 480px):Media features are the conditions that must be met in order to render the CSS within a media query.
  4. rules are nested inside of the media query's curly braces. these rules will be applied only when the media query is met
  5. Range:Specific screen sizes can be targeted by setting multiple width and height media features. min-width and min-height
    @media only screen and (min-width: 320px) and (max-width: 480px) {/* ruleset for 320px - 480px */} media rules only appled between 320px and 480px
  6. can also be targeted by resolution: @media only screen and (min-resolution: 300dpi)
  7. And Operator: can be used to require multiple media features eg. screen size and resolution
  8. Comma Separated List:If only one of multiple media features in a media query must be met, media features can be separated in a comma separated list. eg. @media only screen and (min-width: 480px), (orientation: landscape) so indicates or
  9. Breakpoints:are the screen sizes at which your web page does not appear properly. that when we need media query.
