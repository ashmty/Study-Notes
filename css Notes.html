  7. CSS Grid

    1. Creating a Grid:Needs grid container and grid items,Parent selector {display: grid;} for Block Elements, or {display: inline-grid;}, There are two axes in a grid layout — the column (or block) axis and the row (or inline) axis.
      • By default the Grid have only 1 column
      • to specify the columns and sizes : {grid-template-columns:10 20 30} for 3 columns the value
      • to specify rows and sizes: {grid-template-rows: 10px 20% 30em }
      • to replace both Values above: {grid-template: 200px 300px ( rows ) / 20% 10% 70% ( columns)}
    2. fr: unit to divide available space between rows and columns in this Example 60px out of 100px leaves 40px divided between 2
    3. Repeat: {grid-template-columns: repeat(3, 100px)} this value will repeat it 3 times, so instaed of writting {grid-template-columns: 100px 100px 100px;}
    4. minmax: for the grid part not to go smaller than a specific size {grid-template-columns: 100px minmax(100px, 500px) 100px;} so the first and the third have a specific size while the one in the middle change with the browser
    5. Grid Gap: allow space between the items in the grid, it will not add space at the beginning or end of the grid {grid-row-gap:}or {grid-column-gap} or {grid-gap:rows column} ps. no / inbetween
    6. Multiple Row Items: Row grid lines and column grid lines start at 1 and end at a value that is 1 greater than the number of rows or columns the grid has
      • grid-row-start and grid-row-end where always the value of end is higher than the needed value eg. rows from 2 to 3 = start 2 , end 4
      • shortcute : grid-row { start / end }
      • Span: till the browser to span over 2 cells, {grid-column: 4 / span 2;} start at 4 and join 2 cells
      • Grid-area: shortcute for spanning rows and cloumns for 1 cell ; {grid-area: 2 grid-row-start/ 3 grid-row-end/ 4 grid-column-start / 5 grid-column-end }
    7. Grid-template-areas: allows you to name sections of your web page to use as values in the grid-row-start, grid-row-end, grid-col-start,grid-col-end, and grid-area
      • Each area is called for how many cells in the grid
      • Each item gets a {grid-area: name for each block}
    8. Overlapping Elements: Css grid allows over lapping of cells , just add z-inedx:5 for the top level.
    9. Justify-items: is a property that positions Grid items along the inline, or row, axis. This means that it positions items from left to right across the web page. Demo
      • start — aligns grid items to the left side of the grid area
      • end — aligns grid items to the right side of the grid area
      • center — aligns grid items to the center of the grid area
      • stretch — stretches all items to fill the grid area
    10. Justify Content: position a Grid within its parent element. works when we specify grid width bigger than it's elements.
      1. start — aligns the grid to the left side of the grid container.
      2. end — aligns the grid to the right side of the grid container
      3. center — centers the grid horizontally in the grid container
      4. stretch — stretches the grid items to increase the size of the grid to expand horizontally across the container
      5. space-around — includes an equal amount of space on each side of a grid element, resulting in double the amount of space between elements as there is before the first and after the last element
      6. space-between — includes an equal amount of space between grid items and no space at either end
      7. space-evenly — places an even amount of space between grid items and at either end
    11. Align Items: Demo: position grid items from top to bottom
      1. start — aligns grid items to the top side of the grid area
      2. end — aligns grid items to the bottom side of the grid area
      3. center — aligns grid items to the center of the grid area
      4. stretch — stretches all items to fill the grid area
    12. Align Content: Demo: positions the rows along the column axis, or from top to bottom.
      1. start — aligns the grid to the top of the grid container
      2. end — aligns the grid to the bottom of the grid container
      3. center — centers the grid vertically in the grid container
      4. stretch — stretches the grid items to increase the size of the grid to expand vertically across the container
      5. space-around — includes an equal amount of space on each side of a grid element, resulting in double the amount of space between elements as there is before the first and after the last element
      6. space-between — includes an equal amount of space between grid items and no space at either end
      7. space-evenly — places an even amount of space between grid items and at either end
    13. Justify Self and Align Selfis for the grid child elements. Demo: Justify-self:(left and right) specifies how an individual element should position itself with respect to the row axis, Align Self: specifies how an individual element should position itself with respect to the column axis. works on each individal Block (specific)
      1. start — positions grid items on the left side/top of the grid area
      2. end — positions grid items on the right side/bottom of the grid area
      3. center — positions grid items on the center of the grid area
      4. stretch — positions grid items to fill the grid area (default)
    14. Implicit vs. Explicit Grid: Explicit Grid: predefined defining the dimensions of the grid (stable), Implicit Grid:automatically increase the grid size based on the data demands , The default behavior of the implicit grid is as follows: items fill up rows first, adding new rows as necessary.
      1. Grid Auto Rows and Grid Auto Columns:specifies the height of implicitly added grid rows/columns. Value could be: px,Fr,%, or repeat.
      2. Grid Auto Flow:specifies whether new elements should be added to rows or columns. accepts these values: row , column, dense: attempts to fill holes earlier in the grid layout if smaller elements are added , it can accept 2 values at the same time: {grid-auto-flow: row dense;}
